report_problem 문제 정의 (Problem)

기존 사내 지식 관리 시스템은 파편화된 문서 구조와 느린 검색 성능으로 인해 구성원들이 필요한 정보를 찾는 데 평균 15분 이상을 소요하고 있었습니다.

  • 방대한 데이터 대비 낮은 인덱싱 효율성
  • 사용자 중심의 UI 부재로 인한 낮은 사용률
  • 모바일 환경에서의 접근성 부족

architecture 해결 과정 (Process)

search 01. 사용자 리서치 및 인터뷰 expand_more

현업 종사자 20명을 대상으로 심층 인터뷰를 진행하여 '가장 많이 찾는 정보'의 카테고리를 분류하고 가독성을 저해하는 요소를 파악했습니다.

draw 02. 인터페이스 프로토타이핑 expand_more

노션의 블록 기반 구조를 벤치마킹하여 정보의 계층 구조를 명확히 했습니다. 1px 그리드 시스템과 넉넉한 여백(Whitespace)을 활용하여 시각적 피로도를 최소화했습니다.

code 03. 기술적 최적화 (Indexing) expand_more

ElasticSearch를 도입하여 검색 반응 속도를 2초에서 0.3초 이내로 단축시켰으며, 클라이언트 사이드 캐싱 기술을 통해 빈번하게 접근하는 문서의 로딩 속도를 개선했습니다.
